Actual field case results are presented in Table 1 to show the impact of the U-value on the friction coefficient to match the measured data. When the default U-value is used, the friction coefficients need to be increased to account for the effect of temperature on the dP. With the adjusted U-values, the calculated friction coefficients are closer to 1 compared to default U-values, which increases confidence in the calibrated model’s accuracy.

The same case was run using the existing IPM model with the No Optimisation option. The plant inlet temperatures are provided in Table 2. As expected, the inlet temperatures for the existing model are very low because the default U-value was used. The temperature predictions in the calibrated model were significantly better with an error of only 4°C of the actual plant inlet temperature. The calibrated model can be used for MS routing selection to ensure sufficient crude temperature is entering the plants during colder times of the year to optimize plant throughput

Table 3 illustrates oil production errors for both the calibrated and existing models. The existing model overestimated field production rates by 4% and was found to be unreliable for IPM production forecasts. The calibrated model shows very good agreement with field data and can be used for IPM production forecasts with a high level of confidence.

Whitespace was calculated by opening all wellhead chokes to 100% and using the Optimise with all constraints option. The results for both the calibrated and existing models are provided in Table 4. The error in the calibrated model whitespace forecast was between three and four percent less than the existing model. The results were shared with other IPM subject matter experts and TCO management approved the use of this model for whitespace forecasts. Subsequently, the long term IPM model was rebuilt on top of the short term model by incorporating new wells and facilities.
