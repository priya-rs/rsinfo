IRT, Integrated Reservoir Tool, is an In-house developed application. It is used to visualize the data in master database. It addresses all types of data ranging from daily to monthly to sporadic; from geology to reservoir to production; from static to dynamic data. Dynamic data like daily metering from field, well activities of the rig, production tests on wells; Static data like well header, well marker, trajectories, well logs, well reservoir, well cores, RFT pressures, perforations; Interpretations like core analysis, fluid analysis, survey results, flow test results; Monthly production and injection data - all can be visualised from a single place. Figure 1 shows the tabs available in IRT.

IRT is built using Microsoft Access with linked oracle database. Having linked master database tables enables automatic refresh of data. The data available in master database is immediately visible to the user. This application infrastructure model uses both oracle capability of data integrity and secutiry and Miscrosoft access programming feature. With the latest release of Microsoft access, IRT graphical user interface was redesigned with menu bar and tree-view. This further enhanced user friendly feature of the tool. Figure 2 shows some of the plots available in IRT

The field has different types of producer wells. They are classified as deviated wells, horizontal wells, multilateral wells upto 5 horizontal drains dedicated to either tight layer or dedicated to permeable layers. The global overview of production (Figure 3) is plotted which is a combination of 4 plots. The first is a regular bubble map which shows the location of the well, the size of the bubble shows the cumulative oil production and the color code is for different completion type. The color code is kept consistent in all plots. The second plot shows production rate, watercut and Gas Oil Ratio. The third plot shows cumulative oil production Vs months from startup. The fourth plot is the histogram of cumulative production plotted on the year of startup.

The remaining example throughout the paper shows the data for a realitively small sub layer which has the minimum number of producers and injectors. Figure 4 shows production data for a sub layer. Deviated producers producing from multiple layers and dedicated horizontal producers are identified with different colours. Plot 1 in the dashboard shows the location, type and cumulative oil production of the producers. The other plots follow the same principle as the Global Production overview dashboard.

Figure 5 is a dashboard that links the saturation distribution map with evolution of saturation with time. The color of the points in the saturation evolution with time plot is based on type of tool used to measure saturation data weather it is Open hole tool or different types of cased hole tool. The interactive link between the two plots in the dashboard is useful to observe trends in an area. By selecting the required area in the map the corresponding measurements are highlighted in the saturation vs time plot. This ability helps to quickly observe any trend of saturation evolution in the selected region.

Figure 6 shows the dashboard for pressure evolution analysis and can be used to analyse any sector of the layer. The sealing effects of faults, sectorisation, zone of influence of injectors etc can be analysed by this plot. By introducing the injector startup and shut-in time we can see the effectivity of injection in the required area.
